# Game Specification

1. App Stages

The app has four successive top-level stages:
    login → intro → game → upload


2. Game State Representation

During the game stage, the current unit state is represented by a discrete tuple:
(lid, qcode)

Where:
- lid ∈ {0, 1, 2, 3, 4, 5, 6}
- qcode ∈ {0, 1, 2, 3, ...}

The tuple uniquely identifies the active level and question.


3. CanvasRuntime State Refresh

CanvasRuntime is responsible for responding to in-game state changes.

- A signal listener in useFrame detects the first frame in which the game state changes.
- Upon detection, a StateRefresher is triggered.
- The StateRefresher resets and re-places all relevant 3D objects to ensure a valid starting
  configuration for the new (lid, qcode) state.
